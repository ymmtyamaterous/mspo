# UI / 画面設計書

## 概要

フロントエンドは **React + TanStack Router + Vite** で構成する。  
デザインは `docs/user/sample.html` のエディトリアル・雑誌風スタイルを基準とし、shadcn/ui コンポーネントをカスタムして使用する。

---

## デザインシステム

### カラーパレット

| 変数名 | 値 | 用途 |
|-------|----|------|
| `--ink` | `#1a1410` | テキスト・枠線・ボタン背景 |
| `--paper` | `#f7f3ed` | ページ背景・カード背景 |
| `--accent` | `#e8450a` | 主要アクセント（CTA・タグ・数値） |
| `--accent2` | `#1a6e3c` | サブアクセント（統計数値） |
| `--muted` | `#8a7d6e` | サブテキスト・ラベル |
| `--border` | `rgba(26,20,16,0.15)` | ボーダー・区切り線 |
| `--search-bg` | `#ede8e0` | 検索セクション背景 |

### タイポグラフィ

| 用途 | フォント | サイズ | 備考 |
|------|---------|--------|------|
| 大見出し | Bebas Neue | 44〜72px | letter-spacing 強め |
| セクションタイトル | Bebas Neue | 28〜32px | |
| ロゴ | Bebas Neue | 22px | |
| 本文 | Noto Sans JP | 13〜14px | line-height: 1.8 |
| ラベル・キャプション | Noto Sans JP | 10〜12px | uppercase + letter-spacing |

### ボタン

| 種類 | スタイル |
|------|---------|
| Primary | 背景: `--accent`、文字: white、padding: 12px 24px |
| Secondary（Ghost） | 背景: transparent、枠: `--ink` 1.5px solid |
| Dark | 背景: `--ink`、文字: `--paper` |

### 共通レイアウト原則

- グリッドの境界線は `1.5px solid var(--ink)` で強調
- カードホバー時は `background: rgba(26,20,16,0.04)` で薄く変化
- ナビゲーションは `position: sticky; top: 0` で固定表示
- セクション間はボーダーで明確に区切る

## 画面一覧

### 1. トップページ（`/`）

**目的**: サイトの入口。スポーツへの興味を引きつける。

#### レイアウト構成

```
┌──────────────────────────────────┐
│ NAV（ロゴ・メニュー・検索ボタン）    │ sticky
├──────────────────────────────────┤
│ HERO セクション（2カラム）          │
│  左: キャッチコピー + CTA ボタン    │
│  右: 注目スポーツカード × 2 + 統計  │
├──────────────────────────────────┤
│ TICKER（スポーツ名スクロールアニメ） │
├──────────────────────────────────┤
│ 検索バー                          │
├──────────────────────────────────┤
│ カテゴリグリッド（4列 × 2行）       │
├──────────────────────────────────┤
│ 今週の特集（メイン + サイドカード）  │
├──────────────────────────────────┤
│ FOOTER                           │
└──────────────────────────────────┘
```

#### コンポーネント構成

| コンポーネント | 説明 |
|--------------|------|
| `<NavBar />` | ロゴ、メニューリンク、「スポーツを探す」ボタン |
| `<HeroSection />` | キャッチコピー・CTA・注目スポーツ2件・サイト統計数値 |
| `<SportsTicker />` | スポーツ名を横スクロールするアニメーション帯 |
| `<SearchBar />` | キーワード入力 + 検索ボタン（Enter キー対応） |
| `<CategoryGrid />` | カテゴリカード8件をグリッド表示 |
| `<PickupSection />` | 今週のメイン特集 + サイドカード3件 |
| `<Footer />` | リンク集・コピーライト |

#### 表示データ

- ヒーロー: 「今日の注目」「急上昇中」スポーツ各1件（`pickup.getCurrent` から取得）
- 統計: 総登録スポーツ数・対象国数・今週の新着数（`sports.list` 集計）
- カテゴリ: `categories.list` の全件
- 特集: `pickup.getCurrent` の main + sideCards

### 2. スポーツ一覧ページ（`/sports`）

**目的**: スポーツを検索・絞り込みして探す。

#### レイアウト構成

```
┌────────────────────────────────────────────┐
│ NAV                                         │
├──────────────┬─────────────────────────────┤
│ サイドバー    │ メインコンテンツ               │
│ カテゴリ絞込  │ 検索バー                       │
│ ソート選択   │ スポーツカード一覧（3列グリッド）  │
│             │ ページネーション               │
└──────────────┴─────────────────────────────┘
│ FOOTER                                      │
└─────────────────────────────────────────────┘
```

#### コンポーネント構成

| コンポーネント | 説明 |
|--------------|------|
| `<SportsFilter />` | カテゴリチェックボックス・ソート選択 |
| `<SportsGrid />` | スポーツカードのグリッドレイアウト |
| `<SportCard />` | サムネイル・名前・発祥国・難易度・閲覧数 |
| `<Pagination />` | ページネーションコントロール |

#### `<SportCard />` の表示要素

- カテゴリカラーバー（上部 4px）
- スポーツ名（Bebas Neue、大きめ）
- 発祥国
- 難易度（★の数）
- カテゴリ名
- 閲覧数

#### URL パラメータ

| パラメータ | 説明 |
|-----------|------|
| `?search=` | キーワード検索 |
| `?category=` | カテゴリ ID |
| `?sort=` | `createdAt` / `viewCount` / `popularity` |
| `?page=` | ページ番号 |

### 3. スポーツ詳細ページ（`/sports/:id`）

**目的**: 1つのスポーツの全情報を表示する。

#### レイアウト構成

```
┌────────────────────────────────────┐
│ NAV                                 │
├────────────────────────────────────┤
│ パンくずリスト                       │
├──────────────────┬─────────────────┤
│ メイン（左）      │ サイドバー（右）  │
│ スポーツ名・説明  │ メタ情報         │
│ ルール説明       │ カテゴリ         │
│ 歴史・背景       │ お気に入りボタン  │
│ 動画（あれば）   │ 関連スポーツ     │
└──────────────────┴─────────────────┘
│ FOOTER                              │
└─────────────────────────────────────┘
```

#### 表示要素

**メインエリア**

| 要素 | 説明 |
|------|------|
| スポーツ名（日本語・英語） | Bebas Neue で大きく表示 |
| 概要説明 | マークダウン or プレーンテキスト |
| ルール | セクション区切りで表示 |
| 歴史・背景 | セクション区切りで表示 |
| 動画埋め込み | `videoUrl` が存在する場合 iframe で表示 |

**サイドバー**

| 要素 | 説明 |
|------|------|
| アイキャッチ画像 | `imageUrl` が存在する場合 |
| メタ情報グリッド | 発祥地・競技人口・競技歴・難易度 |
| カテゴリバッジ | クリックでカテゴリ一覧へ遷移 |
| タグ一覧 | `sport_tag` のタグ群 |
| お気に入りボタン | ログイン済みのみ表示 |
| 閲覧数 | ページ訪問時にインクリメント |

**関連スポーツ**（同カテゴリから最大4件）

### 4. カテゴリ一覧ページ（`/categories`）

**目的**: カテゴリ一覧を表示し、各カテゴリへの導線を提供する。

#### レイアウト構成

- 4列グリッドでカテゴリカードを表示
- 各カードにはアイコン（絵文字）・名前・説明・スポーツ件数
- カードクリックで `/sports?category=:id` へ遷移

---

### 5. ランキングページ（`/ranking`）

**目的**: 人気スポーツをランキング形式で表示する。

#### レイアウト構成

```
┌─────────────────────────────────┐
│ NAV                              │
├─────────────────────────────────┤
│ タブ: 閲覧数 | お気に入り数        │
├─────────────────────────────────┤
│ ランキングリスト（順位・名前・数値）│
└─────────────────────────────────┘
│ FOOTER                           │
└──────────────────────────────────┘
```

#### `<RankingItem />` の表示要素

- 順位数字（Bebas Neue、大きめ、`--accent` 色）
- スポーツ名
- カテゴリ名
- 発祥国
- スコア（閲覧数 or お気に入り数）
- 人気度バー（0〜100% の横棒）

---

### 6. スポーツ投稿フォーム（`/submit`）

**目的**: ログインユーザーが新しいスポーツを投稿する。

#### フォーム項目

| フィールド | 入力形式 | 必須 |
|-----------|---------|------|
| スポーツ名（日本語） | テキスト | ✓ |
| スポーツ名（英語） | テキスト | |
| カテゴリ | セレクトボックス | ✓ |
| 概要・説明 | テキストエリア（最大2000文字） | ✓ |
| ルール説明 | テキストエリア | |
| 歴史・背景 | テキストエリア | |
| 発祥国 | テキスト | |
| 発祥年 | 数値入力 | |
| 競技人口 | テキスト（例: 約200万人） | |
| 難易度 | 星評価（1〜5） | |
| アイキャッチ画像 URL | URL 入力 | |
| 紹介動画 URL | URL 入力 | |
| タグ | カンマ区切りテキスト | |

#### UX 要件

- 未ログインの場合はログインページへリダイレクト
- 送信後「審査中」の通知を表示し、マイページへ遷移
- バリデーションエラーはフィールド直下に表示

---

### 7. お気に入り一覧（`/my/favorites`）

**目的**: ログインユーザーのお気に入りスポーツ一覧を表示する。

- `<SportCard />` と同じカードコンポーネントを使用
- お気に入り解除ボタン付き
- 空の場合は「お気に入りがまだありません」メッセージ + 探索ボタン

---

### 8. 自分の投稿一覧（`/my/submissions`）

**目的**: ログインユーザーが自分の投稿を管理する。

#### 表示内容

| 項目 | 説明 |
|------|------|
| スポーツ名 | リンク（公開済みのみ） |
| ステータスバッジ | `pending`（審査中）/ `published`（公開中）/ `rejected`（却下） |
| 投稿日時 | |
| 操作ボタン | 編集（`pending` / `published`）/ 削除（`pending` のみ） |

---

### 9. ログイン（`/login`）・サインアップ（`/signup`）

既存の `sign-in-form.tsx` / `sign-up-form.tsx` コンポーネントをベースに使用。  
デザインをサイト全体のテーマ（クリーム紙背景・Bebas Neue 見出し）に合わせてカスタマイズする。

---

### 10. 管理者ダッシュボード（`/admin`）

**目的**: 管理者がコンテンツを審査・管理する。

#### サブページ

| パス | 機能 |
|------|------|
| `/admin` | ダッシュボード（承認待ち件数・統計） |
| `/admin/submissions` | 承認待ちスポーツ一覧 |
| `/admin/sports` | 公開済みスポーツ管理 |
| `/admin/categories` | カテゴリ管理 |
| `/admin/pickup` | 今週のピックアップ設定 |

## ローディング・エラー状態

| 状態 | 表示内容 |
|------|---------|
| データ取得中 | Skeleton コンポーネントで骨格表示 |
| 取得エラー | エラーメッセージ + 再試行ボタン |
